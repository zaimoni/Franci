Recall that we found that the standard definitions of mathematical collection equality, only fully work for classical logic: reflexivity of equality = does not hold for any of the truth-table describable non-classical logics we are considering. We would like alternate definitions for the non-classical logics that:

  • Are equivalent to the standard definition for classical logic;
  • Respect the intended interpretations of the non-classical logics we are considering; and
  • Fail less, i.e. they evaluate to true or false more than the standard definition does.

I will be explicitly flagging standard terminology here, not non-standard terminology.

Towards a theory of truth

I strongly recommend watching Carnaedes.org's videos on Tarski's theory of truth, and its refinement by Saul Kripke. I'm interpreting these, for the instructions I will be following.

  • The intended interpretation of TruthValued, is the domain of discourse of all syntactically truth-valued expressions. (Whether these actually have truth values, i.e. are statements, depends on the logic applied to them.)
  • The intended interpretation of TruthValues, is the domain of discourse of all truth values for a given logic.
  • The axiomatic identity inference rule A ├ A, may be thought of as evaluating all statements A to themselves. Since we have a structural rule that de-duplicates the hypotheses of an inference rule, this never infers any new statements not already present.
  • As an example of an inference rule that takes a single hypothesis, consider the syntactical equivalence idempotence of &, which summarizes a pair of inference rules. In the reverse direction, this takes an arbitrary statement P and infers P & P. The forward direction is only applicable to statements interpretable as instances of P & P, and infers P.
  • The notation A ↦ true, etc., we introduced, can be thought of as stating that a statement A evaluates to the truth value true, etc. That suggests ↦ should have a domain contained within the domain of discourse all truth valued expressions, and range truth values. Technically, ↦ is taking the logic as an implicit parameter. These are predicates on the domain of discourse of syntactically truth-valued expressions.

TruthValued is not formally describable. It includes things like English sentences, indeed sentences in any natural language.

In contrast, TruthValues is formally describable: the truth values true, false, unknown, and contradiction are unequivocally ur-elements, rather than mathematical objects. For the set theories with ur-elements, we can interpret the principle of bivalence/trivalence/tetravalence, into this new notation.

Replacing logical or ∨ with the existence quantifier ∃, is stronger for Belnap's four-valued logic: it requires (P ↦ t) ↦ true for at least one of the truth values t directly, regardless of the logic applied. (This already was inferable, for the other five truth table describable logics we are considering.) The principle of non-contradiction then ensures that non-classical truth values are never allowed when evaluating P ↦ t . (Recall that truth-table calculations are an alternate notation, for classical logic reasoning about truth-table describable logics.)

If we need to explicitly track which logic TruthValues is from, we can subscript it; e.g., TruthValuesclassical.

As a matter of notation, for an explicitly enumerated mathematical collection such as those mentioned in the table: any term named in the enumeration is in the collection, while any term not listed, is not in the collection. That is, all explicitly enumerated mathematical collections, whose enumeration only has terms (i.e., no variables) are sets, regardless of choice of set theory.

It is tempting to abuse notation yet to be defined and say that:

  • For domains of discourse D,E we say f:D → E has domain D and range E. We say f has type D → E. If we have an explicit definition f(x) := y, we have f(x) ↦ y with intended interpretation "f of x evaluates to y". Alternately, we have x ↦f y with intended interpretation "f of x evaluates to y".
  • The syntactical inference rules summarized by idempotence of & both have type TruthValued → TruthValued. The one that removes logical and &, has an implicit domain restriction; we only define it when the inference rule is applicable.
  • The notation A ↦ true, etc., we introduced, has type TruthValued → TruthValues.

That is, in our overview of logic we have a default f:TruthValued → TruthValues, not mentioned at the time.

  • The intended interpretation of Truth:TruthValued → TruthValues, is a summary of the process that evaluates truth values for syntactically truth-valued statements. There is one of these for each logic we are considering. If we need to track the exact logic, we can subscript; e.g., Truthclassical.
  • For a truth-valued expression P and truth value t, the notations P ↦ t and P ↦Truth t have the same intended interpretation.
  • We expect that if [Q/P] is a valid uniform substitution, then Truth(A)=Truth(A[Q/P]) for any syntactically truth-valued expression A.

I intend Truth to be closely related to Saul Kripke's theory of truth. As such, it does have an implicit domain restriction. It is undefined for any truth-valued expression, for which an in-logic truth value cannot be assigned.

We start with trying to rewrite ⇔ in terms of the new Truth predicate. This requires taking the properties of equality as axioms for truth values. The forward direction works for all of the logics we are interested in, except Belnap's. The reverse direction fails for the non-classical logics on the extra cases introduced by non-classical truth values for the principle of trivalence or tetravalence. (That is, evaluating P ⇔ Q is no longer constant true when the non-classical truth values are introduced.) The overall result is a classical logic only syntactical equivalence:

Truth(P)=Truth(Q) ⟛ P ⇔ Q

Recall that for classical logic only, syntactical equivalences justify substitutions. So we can apply this substitution to any of the four definitions of equality for mathematical collections, to get an alternative definition that is operationally identical, for classical logic only. Since the standard definitions only worked for classical logic, this is a reasonable strategy to get a better definition.

As an alternate definition for the non-classical logics, the classical logic rewrite is not circular: it defines class/set equality in terms of axiomatic equality whose validity is restricted to truth values (that is, we have no axiomatic equality of truth-valued expressions, only of truth values.)

Exercises: This rewrite does make reflexivity of class/set equality unconditional for all five the non-classical, truth table describable, logics we are considering. It also gets us transitivity of equality for Belnap's four-valued logic. Both of these depend on the principle of bivalence/trivalence/tetravalence (i.e., if we have a truth-valued expression that does not have a truth-value, then either the evaluation outright fails, or we extend the evaluation. if that extension uses a non-classical truth value then we can get a non-classical truth value for equality =.)

We consider what we can infer from knowing that x is an element of a mathematical collection A, while y is not an element of A. The "substitution leads to absurdity" approach suffers from the weakness of modus tollens in the presence of non-classical truth values.

The extra clause in the definition of equality for the Zermelo-Frankel set theories (from only having one domain of discourse) allows us to get the classical result in the presence of non-classical truth values, when using the Truth rewrite of the standard definition. This doesn't transparently work for the Von Neumann-Bernays-Gödel set theories.

This feels like a "gap" in our axioms for equality. A proper mathematical model of notation, should let us fix this.
