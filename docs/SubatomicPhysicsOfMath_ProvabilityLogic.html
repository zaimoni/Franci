Following prior art, we consider a specialization of modal logic to the existence (or lack thereof) of proofs. We start by defining what notations are allowed for the simplest case, propositional modal logic:

  • A supply of propositional variables, and a bijection domain ℕ and range propositional variable notations. We say that for a bijection f with domain ℕ, that Range(f) is countably infinite.
  • We assume a base logic (our authorities typically use classical or intuitionistic logic), that provides the inference rules and logical connectives to be used with the propositional variables. This implicitly defines the non-modal truth-valued notations.
  • The modal operator ◻, is a (partial?) function with domain truth-valued notations, and range TruthValues.
  • The modal operator ◊, is tradtionally defined as ~◻~ for classical logic. We expect, by analogy with ∀ and ∃, that this will need adjustment in the presence of non-classical truth values.

What is not obvious a priori, is that this notational system has many intended interpretations. Modal logic is usually introduced as the logic of possibility, with ◻ having intended interpretation "necessarily" and ◊ having the intended interpretation "possibly".

We say that the legal notations implicitly defined by the above, is Notationmodal. It is reasonable that there is a formal grammar corresponding to the above.

Kripke i.e. relational semantics

Rather than take ◻ to be an undefined predicate, we define both ◻, and semantic entailment ╞ as follows:

  • We say a Kripke frame, i.e. relational frame, is an ordered pair (W, R) where W∈Set, R ∈ Function(W2, TruthValues). We say that the elements w of W are nodes, or worlds. R is taken to use infix notation, and is said to be the accessiblity relation.
  • We say a Kripke model is an ordered triple (W, R, ╞) where (W, R) is a Kripke frame, and ╞ ∈ Function(W × Notationmodal, TruthValues). ╞ uses infix notation, like ├. We require that:
    • Truth(w ╞ ~A)=~Truth(w ╞ A)
    • Truth(w ╞ A & B)=Truth(w ╞ A) & Truth(w ╞ B), and similarly for other infix binary logical connectives

These requirements interpret that ╞ is completely determined by its values on propositional variables; this is a structural induction definition. We then define

w ╞ ◻A := (∀u ∈ {x ∈ W : w R x})(u ╞ A)

The classical logic definition of ∃ then yields

Truth(w ╞ ◊ A) = Truth((∃u ∈ {x ∈ W : w R x})(u ╞ A))

To make the non-classical truth value workarounds for ∃ apply as-is here, we define

w ╞ ◊ A := (∃u ∈ {x ∈ W : w R x})(u ╞ A)

Alternate readings of w ╞ A in the non-applied mathematical field of model theory, include "w satisfies A", "A is satisfied in w", or "w forces A". ╞ is also known as the "satisfaction relation", "evaluation", or "forcing relation".

We then say a formula A ∈ Notationmodal is valid in:

  • A model (W, R, ╞) when (∀w ∈ W)(w ╞ A)
  • A frame (W, R) when A is valid for all choices of ╞ (consistent with the current axioms, including those beyond the ones mentioned already)
  • A class C of frames or models, when A is valid for each c∈C.

The definition of validity of A in a frame, is sensitive to set-theoretic axioms; this controls which functions may be said to exist. In general, there are candidates for ╞ that exist for classical logic, but not Intuitionistic logic.

Normal modal logics

Following prior art, we say that if a modal logic satisfies two additional axioms, that it is a normal modal logic:

  • ◻(A ⇒ B) ├ ◻A ⇒ ◻B : Kripke schema, or distribution axiom
  • (├ A) ├ (├ ◻A) : Necessitation rule

Two important intended interpretations of modal logic are typically not normal: deontic logic ("logic of ethics or obligations"), and epistemic logic ("logic of knowledge").

We can conflate a modal logic, with a mathematical collection L of its formulas. For now, assume classical logic. If we require that:

  • All propositional logic tautologies are in L. (More specifically, their notations).
  • All instances of the Kripke schema are in L.
  • L is closed under modus ponens: (P ⇒ Q)∈L, P∈L ├ Q ∈ L
  • L is closed under the necessitation rule

We then define K as the intersection of all such L. This has the same conceptual issues, as proving the natural numbers ℕ are a set from the Axiom of Infinity.

Gödel-Löb logic

We have not yet built out the tools to re-derive Löb's theorem (which is a statement about Gödel number-encoded provability statements by Peano arithmetic, about Peano arithmetic.) The corresponding modal logic axiom is Löb's axiom (schema):

◻(◻P ⇒ P) ├ ◻P

If we include all instances of Löb's axiom in the procedure used to construct K, we end up with what is traditionally defined as GL.

The intended interpretation here, is:

  • ◻P has intended interpretation "It is provable that P"
  • ◊P has intended interpretation "It is consistent that P"

Here, both of the target logic, and the axiom system, are implicit parameters. Ludwig Wittgenstein would have rejected this, for much the same rationale that he rejected the Gödel Incompleteness theorems: to say that no truth value for a formal notation is provable, would mean in his metaphysics that it wasn't even a formal notation even if it was otherwise syntactically valid.

A typical notation for Löb's theorem is:

(PA ├ Prov(P) ⇒ P) ├ (PA ├ P)

"PA" is an abbreviation for "Peano arithmetic"; Prov(P) is a predicate that is true if and only if P has a (Genzten-format) proof in Peano arithmetic. We have not done the required work for Godel's incompleteness theorems and Gentzen's relative consistency proof of Peano arithmetic, but Prov(P) is closely related to the predicate used in those theorems.

So, it looks like if we are using classical logic and the Peano arithmetic axioms to define ◻, that we have both:

  • Truth(◻P) = Truth(PA ├ P)
  • Truth(◻P) = Truth(Prov(P))

So it appears Prov(P) is just (PA ├ P), so Löb's axiom directly interprets Löb's theorem: "If it is provable, that if P is provable then P, then it is provable that P". (There is more going on here; technically we have a predicate Prov(n) with domain ℕ. A Gödel encoding with domain Peano arithmetic statements, and range ℕ is used to define Prov(P) from Prov(n).)

The necessitation rule, here interprets as "an axiom is self-proving". (The Gentzen-format proof of an axiom is one line: the axiom itself.)

We suppose that a formal system can be axiomatized as an ordered triple (grammar, logic, axioms) where the extra-logical axioms itself is a set of Gentzen-format

  • axioms ├ A, and
  • axiom schemata t0∈D0, ... tn∈Dn ├ A(t0, ..., tn), where all term variables in the hypothesis are used in the consequence.

Since the formal grammar both must be the same for both logic, axioms, and often is the most verbose part of the triple, we often elide this parameter and just refer to (logic, axioms). In contrast, we are not as finicky regarding how the logic is described; both truth table, and axiomatic, descriptions are acceptable. Then we take (logic, axioms) to be a parameter for both ◻ and ◊. If we need to explicitly notate this parameter, we use subscripting: ◻(logic, axioms) and ◊(logic, axioms)

Two common abbreviations in the literature are PA for Peano arithmetic (classical logic, Peano axioms), and HA for Heyting arithmetic (Intuitionistic logic, Peano axioms).

We define the following technical English for two-valued logics:

  • P is independent of (logic, axioms) := ◊(logic, axioms)P & ◊(logic, axioms)~P
  • (logic, axioms) is consistent := (∀P ∈ Notation)(~(◻(logic, axioms)P & ~◻(logic, axioms)P))
  • (logic, axioms) is complete := (∀P ∈ Notation)(◻(logic, axioms)P ∨ ~◻(logic, axioms)P)

Defining inconsistent and incomplete formal systems, and extending these definitions to handle non-classical truth values, can be done with the techniques we have already used.

We will not try to prove Gödel's First Incompleteness theorem at this time, but we can informally state its modern refinement now: any formal system (logic, axioms) that models "enough of" Peano arithmetic (or Heyting arithmetic) to encode its proof existence as arithmetic, is at least one of inconsistent or incomplete.

We note that:

  • If P is independent of (logic, axioms), then both (logic, axioms ∪ {P}) and (logic, axioms ∪ {~P}) are formal systems that strictly extend (logic, axioms).
  • If ◻(logic, axioms)P0, then Truth(◻(logic, axioms)P)=Truth(◻(logic, axioms ∪ {P0})P). That is, the extended formal system doesn't prove any new statements.
  • If ◻(logic, axioms)~P0, then (logic, axioms ∪ {P0}) is inconsistent.

We recall that Intuitionistic logic rejects the Law of the Excluded Middle. From the canonical formalization of the Heyting rationale, it is reasonable that the real problem is existential quantification without an evaluation rule. We should have, for classical logic,

(∃t ∈ {true, false})(P ↦ t) ⟛ P ∨ ~P

This is a special case of a general technique of rewriting universal and existential quantification for finite domains of discourse. A weak natural induction proof against notation gives:

All six truth-table describable logics we are considering
(∀x ∈ {x0, ..., xn})P(x) ⟛ P(x0) & ... & P(xn)
Classical, Kleene strong three-valued logic, Franci
(∃x ∈ {x0, ..., xn})P(x) ⟛ P(x0) ∨ ... ∨ P(xn)

The existential rewrite fails for Kleene weak three-valued logic and Lisp/Prolog logic due to unknown ∨ true ↦ unknown. The failure of the rewrite for Belnap's four-valued logic, is due to unknown ∨ contradiction ↦ true.

The Heyting rationale predates Gödel-Löb logic, in the literature, by several decades. We interpret the rejection of the Law of the Excluded Middle, as

~◻(├ (∃t ∈ {true, false})(P ↦ t))

i.e.

~◻(├ P ∨ ~P)

~◻(P ∨ ~P)

Alternate proof by cases

We next consider how to prove that the double negation removal inference rule,

~~P ├ P

is invalid for Intuitionistic logic. It would suffice if the Glivenko's theorem translation of the Law of the Excluded Middle,

~~(P ∨ ~P)

were provable in Intuitionistic logic. Keeping in mind that ~A := A ⇒ ⊥ for Intuitionistic logic, I construct the proof backwards.

We start by inferring a more useful formulation, of the Intuitionistic proof by cases inference rule. From this, we derive an Intuitionistic tautology, whose specialization with absurdity ⊥ gives the Glivenko translation of the Law of the Excluded Middle. (We do not go into details here. Glivenko's theorem states that any classical propositional logic statement Q is provable, if and only if the Intuitionistic propositional logic statement ~~Q is provable. This requires modification in the presence of quantifiers.)

The principle of explosion, makes the formal logics we have been reviewing unsuitable for reasoning with self-contradictory premises. The key idea for paraconsistent logic, is simply not having the Principle of Explosion as an inference rule. We interpret this in Gödel-Löb logic as

~◻(├ false ⇒ Q)

While in our initial truth-table motivated review of logic this was computable, it is possible to get similar results with just inference rules. For the moment, we ignore that a never-true statement is illegal as a hypothesis. When interpreting natural language into symbolic notation, we may very well not know that we are trying to interpret a never-true statement.

Recall (from our overview of logic), that logical and & is closely related to the working set of facts, for the Rete algorithm. So the hypothesis P & ~P represents a self-contradictory knowledge (or belief) that P evaluates to both true, and false.

Generally speaking, we very much want modus ponens, and the conjunction elimination and introduction inference rules. We also want the syntactical entailment introduction rule, and its cognate inference rule for classical logic, implication introduction. So, if the above classical logic proof is to be invalid for a paraconsistent logic, one of the two remaining inference rules must itself be invalid. (This is an active research topic, so there is not yet a consensus on how this is to be done.) More explicitly, it is incorrect in meaning to or-eliminate the proposition used to create the logical or statement itself. We just cannot represent that, in the truth-functional logics we have been considering.

This is closely related to the Gettier paradox in epistemic logic (the logic of knowledge, yet another application of modal logic). We see, from our authorities, that the canonical technique for constructing an instance of the Gettier paradox, is to compute a true logical or statement from a false premise (believed to be true), by disjunction introduction where the introduced statement is true. For example:

Imagine that we are seeking water on a hot day. We suddenly see water, or so we think. In fact, we are not seeing water but a mirage, but when we reach the spot, we are lucky and find water right there under a rock. Can we say that we had genuine knowledge of water? The answer seems to be negative, for we were just lucky.
—(Indian philosopher Dharmottara, c. 770 AD/CE)

If we do not want to interpret how we think into formal notation, another option in the literature is to discard conjunction introduction:

~◻(A, B ├ A & B)

This was first developed as discussive logic, by Jaśkowski student of Lukasiewicz. Jaśkowski's intended interpretation, was a conversation between individuals who are individually self-consistent, but whose beliefs are incompatible.

Metaphysics of formal systems

Reviewing what we have just seen: an ur-element set theory using only classical truth values, can mathematically model the notation for Gentzen-format formal systems directly, as an ordered pair (logic, axioms) using a common formal grammar. This relies on finite sequences of textual glyphs (which are indexed by intervals in the natural numbers ℕ). Furthermore, we have examples of logics that require Gentzen-format notation to formalize, as the natural language rationales are not precise enough: Intuitionistic i.e. constructive logic, and the paraconsistent logics.

So, my first impression is that while both constructive logic, and the paraconsistent logics, are interesting for understanding the rationale, that they themselves are "heavy" compared to truth-table describable logics for formalizing set theory.

On what grounds can we say that there is something "beyond the notation"? E.g.

  • Are the properties of classical logic, dependent on how classical logic is described? Does it matter whether the description is in natural language, or Gentzen format?
  • Are the properties of ℕ, dependent on the notation used to formalize ℕ?
  • In particular: does the choice of textual glyphs matter, when formalizing the above using Gentzen-format notation?

For the last point, the usual approach in higher mathematics, is to interpret a Caesar cipher for formal systems. We start by enumerating the textual glyphs actually used in the formal grammar for (grammar, logic, axioms). In practice, this is a finite set {g1, ..., gn}. We then consider injections from {g1, ..., gn} into textual glyphs

encode0:{g1, ..., gn} → Ur

We already verified that the range restriction encode0 ▷ Range(encode), is a bijection.

For all natural numbers n, we then recursively define encoden+1, for finite sequences in the domain of encoden, as

encoden+1((wk)i=0..m) := (encoden(wk))i=0..m
encode := ∪n ∈ ℕ encoden for x ∈ ∪n ∈ ℕ Domain(encode)
encode((wk)i=0..m) := (encode(wk))k=0...m if not defined above, and encode(wk) is defined for all k ∈ 0...m

The second definition, views a function as a subset of the Cartesian product of its domain and range.

That is: given a formal system (grammar, logic, axioms), and any bijection encode0 whose domain is the textual glyphs used to notate the formal grammar, and whose range is in textual glyphs, we can use encode to simultaneous-substitute all of the textual glyphs, computing a formal system (encode(grammar), encode(logic), encode(axioms)). Furthermore, encode also will translate Gentzen-format proofs for (grammar, logic, axioms), into Gentzen-format proofs for (encode(grammar), encode(logic), encode(axioms)).

This leads to the following tacit metaphysical assumption, underlying all of (mainstream) mathematical practice:

(grammar, logic, axioms) and (encode(grammar), encode(logic), encode(axioms)), formally define the same formal system.

That is, a simultaneous substitution of the textual glyphs, does not change the formalization of anything that contains no textual glyphs. If, in addition, our sense-experience is correct that textual glyphs are not mathematical objects, then:

  • The choice of textual glyphs in formalizing a set theory without ur-elements, cannot change either the set theory, or any mathematical object defined in that set theory.
  • In particular, since both Peano arithmetic (classical logic ℕ) and Heyting arithmetic (Intuitionistic logic ℕ) can be formalized without using ur-elements, the textual glyphs used to formalize these arithmetics does not change what is being formalized.

This makes a mathematical irrealist description, in technical philosophical English, so difficult that some (in particular, Wittgenstein after his mentoring by Brouwer) simply outright reject the above metaphysical assumption. (Recall that a mathematical irrealist thinks any particular mathematical object does not exist, before it is described. Yet if the usual practice of mathemticians is correct, it doesn't matter which textual glyphs are used in the formal description! Any choice of glyphs that formalizes both classical logic and the (second-order) Peano axioms will describe Peano arithmetic, and likewise for Heyting arithmetic.)
